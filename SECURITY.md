# Politique de sécurité

## Signaler une vulnérabilité

Si vous découvrez une vulnérabilité de sécurité, veuillez **NE PAS** créer une issue publique. Contactez-nous plutôt à : [votre-email@example.com]

## Bonnes pratiques

- Ne jamais commiter de clés API ou de secrets dans le code
- Utiliser des variables d'environnement pour toutes les informations sensibles
- Vérifier que `.env` et `.env.local` sont dans `.gitignore`
